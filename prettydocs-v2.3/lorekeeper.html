Set up Local

What is local

For our purposes, Local refers to your personal computer. Basically, setting up Lorekeeper offline so you can try it out and get familiar with the application. This is also necessary if you plan to customize Lorekeeper in any way, such as by changing the theme or CSS, or adding extensions.

You should always test your changes in Local first, and then once you are satisfied with the results and there are no errors, push them to your live server (online).

There are a few applications needed to get things running. Setting up a local version of Lorekeeper is a good way to see how comfortable you feel working with these tools, or whether you would be better suited to hire someone else for setup.

GitHub

Set up a free account on GitHub.

Go to the Lorekeeper repository.

Click fork.

This will put a copy of Lorekeeper in your Github that you can freely modify without affecting the original.

GitKraken

Download GitKraken here.

Go through the default installation and setup, and sign into your GitHub account.

Create a new workspace.

Fill out the required information. Select the Lorekeeper repository you forked.

Click on clone repository and save it to your computer.

XAMPP

Download XAMPP here. Make sure to download the 7.4 version

Launch the application as an administrator and proceed with the default installation, with the following components:

Click Config.

Check Apache and mySQL so they start when you launch XAMPP automatically.

Composer

Install Composer.

Use the default settings and paths.

Open your Command Prompt as an administrator.

By default, you will usually be in your system folder.

(c) Microsoft Corporation. All rights reserved.

C:\WINDOWS\system32>

Go into the directory where you set up your local version of Lorekeeper by typing the following command (without the brackets).

cd [path]

For example, if I put Lorekeeper in my Documents folder I would type the following:

cd C:\Documents\Lorekeeper

Now type:

composer install

This will initiate the installation process. Keep the command prompt open.

PHPmyAdmin

While that is running, go to the following address in your internet browser:

http://localhost/phpmyadmin

Click on New.

Create a database.

.env

In the Lorekeeper directory, create a new file named .env

You may get a warning, just click Yes

Copy the following information into the file:

APP_NAME=Lorekeeper
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost

CONTACT_ADDRESS=
DEVIANTART_ACCOUNT=

LOG_CHANNEL=stack
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=**DATABASE NAME**
DB_USERNAME=root
DB_PASSWORD=

BROADCAST_DRIVER=log
CACHE_DRIVER=file
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

MAIL_DRIVER=
MAIL_HOST=
MAIL_PORT=587
MAIL_USERNAME=
MAIL_PASSWORD=
MAIL_FROM_ADDRESS=
MAIL_FROM_NAME=

AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=
AWS_BUCKET=

PUSHER_APP_ID=
PUSHER_APP_KEY=
PUSHER_APP_SECRET=
PUSHER_APP_CLUSTER=
MIX_PUSHER_APP_KEY="${PUSHER_APP_KEY}"
MIX_PUSHER_APP_CLUSTER="${PUSHER_APP_CLUSTER}"

Put your database name next to DB_DATABASE= and save the file

Install Lorekeeper

Return to the command prompt. Once it has finished installing Composer, run the following commands in sequence:

php artisan key:generate
php artisan migrate
php artisan add-site-settings
php artisan add-text-pages
php artisan copy-default-images
php artisan setup-admin-user

The last command will have you set up your admin user. Be sure to keep your username and password, as you will need these to log into the site and use the admin panel.

During the admin user set up, choose Yes for whether this is a local setup, and Yes for verifying the account automatically.

Now your local version of Lorekeeper should be set up! To launch it, use the following command

php artisan serve

Then go to the address in your web browser

Congratulations! You've set up Lorekeeper on Local.
